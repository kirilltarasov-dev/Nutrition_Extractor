  ⚠️ КРИТИЧЕСКАЯ ПРОБЛЕМА: Frontend запускается как Backend
═══════════════════════════════════════════════════════════════════

ПОЧЕМУ ОШИБКА:
Railway запускает Python команду (uvicorn) 
вместо Node.js команды (npm)

Это значит:
- Либо вы настраиваете backend сервис
- Либо frontend сервис НАСТРОЕН НЕПРАВИЛЬНО

═══════════════════════════════════════════════════════════════════

✅ ЧТО СДЕЛАТЬ:

ВАРИАНТ 1: Удалить Custom Start Command (РЕКОМЕНДУЕТСЯ)

1. В Settings → Deploy → Custom Start Command
2. УДАЛИТЕ всю команду `cd backend && uvicorn...`
3. Оставьте поле ПУСТЫМ
4. Сохраните

Railway автоматически найдет package.json и запустит npm start

═══════════════════════════════════════════════════════════════════

ВАРИАНТ 2: Правильная команда для Frontend

Если нужно указывать вручную:

Custom Start Command:
  npx serve -s build -l $PORT

Pre-deploy Command:
  npm install && npm run build

Providers:
  ✓ Node (должен быть только Node!)
  ✗ Python - УДАЛИТЬ если есть

═══════════════════════════════════════════════════════════════════

🔍 КАК ПРОВЕРИТЬ:

1. Providers должен быть ТОЛЬКО "Node" (JavaScript)
2. НЕ должно быть Python в Providers
3. Custom Start Command НЕ должен содержать "backend" или "uvicorn"
4. Root Directory = "frontend"

═══════════════════════════════════════════════════════════════════

💡 ЕСЛИ НИЧЕГО НЕ ПОМОГАЕТ:

1. Удалите этот сервис
2. Создайте НОВЫЙ сервис для Frontend
3. При создании выберите:
   - GitHub Repo
   - Root Directory: frontend
   - Providers: Node (только!)
4. НЕ добавляйте Custom Start Command
5. Railway сам все определит
