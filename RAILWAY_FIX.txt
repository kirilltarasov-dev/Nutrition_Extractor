  🔧 ИСПРАВЛЕНИЕ: npm: command not found
═══════════════════════════════════════════════════════════════════

ПРОБЛЕМА:
В настройках Frontend сервиса указана BACKEND команда!

Custom Start Command показывает:
  cd backend && uvicorn app.main:app...
  
Это команда для Python, а не для Node.js!

═══════════════════════════════════════════════════════════════════

✅ РЕШЕНИЕ:

1. Найдите в настройках поле "Custom Start Command"
2. УДАЛИТЕ команду `cd backend && uvicorn...`
3. Оставьте поле ПУСТЫМ (или введите для frontend):
   npx serve -s build -l $PORT
   
4. Pre-deploy Command должен быть:
   npm install && npm run build
   
5. Providers: должен быть только Node
6. Root Directory: frontend

═══════════════════════════════════════════════════════════════════

💡 ИЛИ проще:

Удалите вообще весь Custom Start Command.
Railway сам определит Node.js и запустит через npm start
из package.json.
